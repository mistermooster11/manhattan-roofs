# Clone UI Flow — Index

> **Dành cho AI coding agent:** Bộ tài liệu đầy đủ để convert bất kỳ website nào sang Next.js App Router.
> Trước khi viết bất kỳ code Next.js nào, đọc bundled docs tại `node_modules/next/dist/docs/` — đây là source of truth.
> Tham khảo: https://nextjs.org/docs/app/guides/ai-agents

## Quick Start cho AI Agent

Khi nhận task clone một website mới, thực hiện theo đúng thứ tự:

| Bước | Việc cần làm | Tài liệu |
|---|---|---|
| 1 | Download source, lập bản đồ sections, extract colors | `steps/step-1-download-and-prepare.md` |
| 2 | Khởi tạo Next.js, ghi color tokens vào `globals.css` | `steps/step-2-project-setup.md` |
| 3 | Đọc tất cả 6 rules trước khi viết dòng code đầu tiên | `rules/` |
| 4 | Lặp lại cho mỗi section: rà soát codebase → context → viết → kiểm tra | `steps/step-3-convert-section.md` |
| 5 | Chạy checklist hoàn thành | `steps/step-4-final-checklist.md` |

---

## 6 Rules bắt buộc — tóm tắt

| # | Rule | File |
|---|---|---|
| 1 | Thêm `"use client"` khi có state / event / browser API | `rules/rule-1-use-client.md` |
| 2 | `<Image>` cho ảnh local/whitelisted — `<img>` cho CDN ngoài | `rules/rule-2-image.md` |
| 3 | Tailwind class ưu tiên — `style={{}}` chỉ là ngoại lệ | `rules/rule-3-tailwind.md` |
| 4 | `<Link>` cho route nội bộ — `<a>` cho URL ngoài | `rules/rule-4-link.md` |
| 5 | Parent gọn — con độc lập (tách file khi > 40 dòng JSX) | `rules/rule-5-component-architecture.md` |
| 6 | Không hard-code màu / font — dùng CSS variables | `rules/rule-6-fonts-colors.md` |

## Prompt template để giao việc convert section

```
Dựa vào file html, css, js tại downloaded\<domain>, hãy convert section
[TÊN SECTION] sang components/custom/[TênComponent]/[TênComponent].tsx

Trước khi viết:
- Đọc docs/flow-clone/steps/step-3-convert-section.md
- Áp dụng tất cả 6 rules trong docs/flow-clone/rules/
- Tham khảo docs/flow-clone/patterns/common-patterns.md nếu cần

Sau khi viết xong, chạy 4 bước kiểm tra:
  A. get_errors → fix cho đến 0 errors
  B. npx tsc --noEmit → fix cho đến output sạch
  C. get_terminal_output (nếu dev server đang chạy)
  D. Import vào page.tsx → get_errors(page.tsx)
```
