## Workflow for Each Section

### Step 1 — Read Original HTML

```bash
cat "downloaded/csielectric.com/<page>/index.html"
```

Find the section. Note:
- Exact HTML tag + class names (WordPress adds `.wp-block-*`, `.alignfull`, `.is-style-*`)
- Nesting depth: wrapper → inner → content → elements
- Data attributes used for JS (e.g. `data-aos`, `data-target`, `data-count`)
- Any inline `style=""` on elements

### Step 2 — Find the CSS

For a Gutenberg block, the CSS file is named after the block:
```bash
cat "downloaded/.../gutenberg/blocks/<block>/style.css"
```

For general page styles:
```bash
grep -n "<selector>" "downloaded/.../style.css@ver=...css"
```

Note all:
- Base styles (font, color, spacing, background)
- Hover / focus transitions (look for `transition:`, `:hover`, `:focus`)
- Pseudo-elements (`::before`, `::after` overlays)
- Responsive breakpoints (`@media (min-width:…)`)
- Animation classes (`.aos-animate`, `.animated`, `.is-visible`)

### Step 3 — Compare with Clone Component

Open the matching component in `components/custom/<name>/` and its CSS module in `styles/pages/` or `styles/components/`.

Ask:
1. Does the HTML nesting match? (CSS specificity depends on this)
2. Are all class names mapped to CSS Module equivalents?
3. Are pseudo-elements correct? (`::before` overlays need `position:relative` on parent)
4. Are hover chains correct? (`.parentLink:hover .childEl` — the hover must be on the wrapper `<a>`)
5. Are `transition` values identical? (duration, easing, property)
6. Are CSS custom properties used correctly? (`var(--csi-accent)` not hardcoded `#ff6d00`)

### Step 4 — Fix

- Fix HTML structure in `.tsx` first (nesting, wrapper elements)
- Then fix CSS module (selector chains, transitions, pseudo-elements)
- Run `npm run build` after each section to catch CSS Modules errors early

### Step 5 — Test

| Check | How |
|---|---|
| Desktop layout | Browser at 1440px |
| Tablet layout | DevTools 768–1024px |
| Mobile layout | DevTools 375–767px |
| Hover effects | Mouse over each interactive element |
| Focus visible | Tab through with keyboard |
| Animations | Scroll into view, reload page |

## CSS Modules — Common Gotchas

### ❌ Bare element selectors are rejected

```css
/* WRONG — CSS Modules will error */
address { font-style: normal; }

/* CORRECT — scope under a local class */
.locationsList address { font-style: normal; }
```

### ❌ Hover chain needs the hover on the right element

```css
/* WRONG — .card:hover won't work if the <a> wraps the card */
.card:hover .heading { width: 120%; }

/* CORRECT — hover is on the interactive wrapper */
.memberLink:hover .memberHeading { width: calc(85% + 20px); }
```

And the JSX must actually have the wrapper:
```tsx
// WRONG — bare div, no hover trigger
<div className={styles.card}>
  <h2 className={styles.heading}>…</h2>
</div>

// CORRECT — <a> wrapper enables the :hover chain
<a className={styles.memberLink} href={href}>
  <h2 className={styles.heading}>…</h2>
</a>
```

### ❌ Pseudo-elements need position:relative on parent

```css
/* parent must be positioned */
.csiLocations {
  position: relative;  /* ← required */
}
.csiLocations::before {
  content: '';
  position: absolute;
  z-index: 1;
  /* … */
}
```

### ❌ z-index stack needs all layers declared

If you have: background iframe → gradient overlay → text content, declare all three:
```css
.acfMap      { z-index: 0; position: absolute; }
.section::before { z-index: 1; position: absolute; }
.innerContent { z-index: 2; position: relative; }
```

## Animation Patterns in the Original

### AOS (Animate On Scroll)
Original HTML has `data-aos="fade-up"` (or `fade-left`, `zoom-in`, etc.) on elements.

In the clone: use the `AosInit` client component (already set up in layout) + add `data-aos` attributes to JSX elements.

```tsx
<div data-aos="fade-up" data-aos-delay="100">…</div>
```

Check `downloaded/.../js/aos.js` or look for `data-aos` attributes in the original HTML to find which animation each element uses.

### CountUp (number counters)
Original HTML: `<span data-target="150" data-count>0</span>` triggered by IntersectionObserver.

In the clone: use a `useCountUp` hook with `IntersectionObserver`. Check `components/custom/home/StatsSection.tsx` for the existing pattern.

### Typed.js (typewriter text)
Original HTML: `<span class="typed" data-typed-items="item1,item2">` with `typed.min.js`.

In the clone: use the `react-typed` package or a manual `useEffect` with `Typed` constructor.

### Slick Carousel → Manual Slider
Original uses `slick-slider` with dots + arrows. In the clone: use CSS scroll-snap or a simple `useState` index with prev/next buttons.

## Comparing Fonts

All font references in the clone should use CSS variables, not raw font-family strings:

| Variable | Font | Usage |
|---|---|---|
| `var(--font-primary)` | system / fallback | Body text |
| `var(--font-secondary)` | `trade-gothic-next-compressed` | Headings, badges |
| `var(--font-tertiary)` | `trade-gothic-next` | Sub-headings, nav |

If a section uses the wrong font, check what class the original applies and which CSS rule sets `font-family` there.

---

## Comparing Colors

| Variable | Hex | Usage |
|---|---|---|
| `var(--csi-accent)` | `#ff6d00` | Orange CTA, highlights |
| `var(--csi-secondary)` | `#005caf` | Blue headings, badges |
| `--csi-dark` | `#00284c` | Dark navy background |
| white | `#ffffff` | Text on dark backgrounds |

Never hardcode hex values in component CSS — always use the variable.

## Quick Diff Command

To compare a block CSS file line-count vs your module:

```bash
wc -l "downloaded/.../blocks/<name>/style.css"
wc -l "styles/pages/<name>.module.css"
```

If the original is much longer, you are likely missing rules.

To search for a specific property in the original CSS:

```bash
grep -n "transition" "downloaded/.../blocks/leadership/style.css"
grep -n "hover" "downloaded/.../style.css@ver=...css"
```
